CSS Notes

CSS, or Cascading Style Sheets, is a stylesheet language used to control the presentation of HTML documents. It allows you to specify how elements on a web page should be displayed, including aspects like layout, colors, fonts, spacing, and overall visual aesthetics. Here are some key points about CSS:

Separation of Content and Presentation: CSS separates the content (HTML) from its presentation, making it easier to manage and maintain web pages.

Selectors: CSS uses selectors to target HTML elements. For example, you can select elements by their tag name, class, or ID.

Properties and Values: Each CSS rule consists of properties and values. For example, color: blue; sets the text color to blue.

Cascading and Inheritance: The term "cascading" refers to how styles can be overridden. Styles applied later in the stylesheet take precedence over earlier ones. Inheritance allows child elements to inherit styles from their parent elements.

Responsive Design: CSS supports responsive web design, allowing web pages to adapt to different screen sizes and devices through media queries.

Frameworks and Preprocessors: CSS frameworks (like Bootstrap) and preprocessors (like SASS or LESS) can help streamline development by providing reusable styles and features.



CSS Selectors

Selectors are patterns used to select the elements you want to style. Here are the main types of CSS selectors:

  • Universal Selector (*): Selects all elements. * { margin: 0; padding: 0; }
  • Type Selector: Selects all elements of a given type. p { color: blue; } selects all <p> elements.
  • Class Selector (.): Selects elements with a specific class attribute. .highlight { background-color: yellow; }
  • ID Selector (#): Selects a single element with a specific ID. #header { font-size: 24px; }
  • Attribute Selector: Selects elements based on an attribute or attribute value. input[type="text"] { border: 1px solid #ccc; }
  • Child Selector (>): Selects direct children of a specified element. ul > li { list-style-type: none; }
  • Descendant Selector (space): Selects elements that are descendants of a specified element. div p { color: red; }
  • Adjacent Sibling Selector (+): Selects an element that is directly after a specified element. h1 + p { margin-top: 0; }
  • General Sibling Selector (~): Selects all siblings after a specified element. h1 ~ p { color: green; }

Specificity

Specificity determines which CSS rule is applied when multiple rules target the same element. It’s calculated based on the types of selectors used in a rule. Here’s how specificity works:

  • Calculating Specificity: Each selector type has a weight:
    • Inline styles: 1,0,0,0
    • IDs: 0,1,0,0
    • Classes, attributes, and pseudo-classes: 0,0,1,0
    • Type selectors and pseudo-elements: 0,0,0,1
    Combine the weights to calculate total specificity.
  • Examples:
    • #header (ID) → specificity = 0,1,0,0
    • .menu (class) → specificity = 0,0,1,0
    • div p (type) → specificity = 0,0,0,2 (1 for div + 1 for p)
  • Applying Specificity: If two rules conflict, the one with the higher specificity takes precedence. In cases of equal specificity, the last rule declared in the CSS file is applied.
  • Importance of Specificity: Understanding specificity helps avoid conflicts and ensures the intended styles are applied correctly. Use more specific selectors when necessary but strive for maintainable CSS by avoiding overly specific rules.


CSS Box Model

The CSS Box Model is a fundamental concept that defines how elements are structured and displayed on a web page. Each element can be considered as a box, which consists of several components:

  • Content: The innermost part of the box, where text and images appear. Dimensions are defined by the width and height properties.
  • Padding: The space between the content and the border. Adds space inside the box, affecting the total size of the box. Controlled by the padding property (e.g., padding: 10px;).
  • Border: Surrounds the padding (if any) and content. Can be styled with width, color, and style (e.g., solid, dashed). Controlled by the border property (e.g., border: 1px solid black;).
  • Margin: The outermost space around the box, creating distance from other elements. Affects the spacing between boxes. Controlled by the margin property (e.g., margin: 20px;).

Box Model Calculation

The total width and height of a box can be calculated using the following formulas:

  • Total Width = width + padding-left + padding-right + border-left + border-right + margin-left + margin-right
  • Total Height = height + padding-top + padding-bottom + border-top + border-bottom + margin-top + margin-bottom

Layout Techniques

Various layout techniques can be used to position elements on a web page effectively:

  • Normal Flow: Elements are arranged in the order they appear in the HTML, from top to bottom, left to right.
  • Float: The float property allows elements to be taken out of the normal flow and positioned to the left or right. Commonly used for text wrapping around images. Example: float: left;
  • Positioning: The position property controls how elements are positioned in the layout:
    • Static: Default positioning; elements are positioned according to the normal flow.
    • Relative: Positioned relative to its normal position (using top, right, bottom, left).
    • Absolute: Positioned relative to the nearest positioned ancestor (not in the normal flow).
    • Fixed: Positioned relative to the viewport, remains in place during scrolling.
    • Sticky: A hybrid of relative and fixed positioning, becomes fixed after a specified scroll point.
  • Flexbox: A layout model that allows for flexible and responsive designs. Uses display: flex; on a container to create a flexible box layout. Aligns and distributes space among items along a single axis.
  • Grid: A powerful two-dimensional layout system that uses rows and columns. Uses display: grid; on a container, allowing for complex layouts with ease. Items can span multiple rows and columns.
  • CSS Frameworks: Frameworks like Bootstrap or Foundation provide predefined grid systems and components to facilitate responsive design.


Flexbox (Flexible Box Layout)

Flexbox is a one-dimensional layout model that allows for easy alignment and distribution of space among items in a container. It is particularly useful for aligning items in a row or column.

Key Concepts

  • Flex Container: A parent element that contains flex items. To make an element a flex container, apply the display: flex; or display: inline-flex; property.
  • Flex Items: The direct children of a flex container. Flex items can be manipulated using various flex properties.
  • Main Axis and Cross Axis: The main axis is the primary direction along which flex items are laid out (horizontal by default). The cross axis is perpendicular to the main axis (vertical by default).

Flexbox Properties

  • Container Properties:
    • flex-direction: Defines the direction of the main axis. Values: row, row-reverse, column, column-reverse.
    • justify-content: Aligns flex items along the main axis. Values: flex-start, flex-end, center, space-between, space-around.
    • align-items: Aligns flex items along the cross axis. Values: flex-start, flex-end, center, baseline, stretch.
    • flex-wrap: Controls whether flex items should wrap onto multiple lines. Values: nowrap, wrap, wrap-reverse.
  • Item Properties:
    • flex-grow: Defines the ability of a flex item to grow relative to others. Default is 0 (no growth).
    • flex-shrink: Defines the ability of a flex item to shrink relative to others. Default is 1 (can shrink).
    • flex-basis: Defines the initial size of a flex item before space is distributed. Can be a length value or auto.
    • align-self: Overrides the align-items property for individual flex items.

Example


        .container {
            display: flex;
            flex-direction: row;
            justify-content: space-between;
            align-items: center;
        }

        .item {
            flex: 1; /* Allows items to grow and fill available space */
        }

Grid Layout

Grid is a two-dimensional layout model that allows for the arrangement of elements in rows and columns, enabling more complex layouts than Flexbox.

Key Concepts

  • Grid Container: A parent element that defines a grid layout. To create a grid container, apply the display: grid; or display: inline-grid; property.
  • Grid Items: The direct children of a grid container. Grid items are placed into the grid cells.
  • Grid Lines: Horizontal and vertical lines that divide the grid into cells. They can be referenced by their numerical index.

Grid Properties

  • Container Properties:
    • grid-template-columns: Defines the number and size of columns in the grid.
    • grid-template-rows: Defines the number and size of rows in the grid.
    • grid-gap (or gap): Sets the size of the gap between rows and columns.
    • grid-template-areas: Allows naming grid areas for easier placement of items.
  • Item Properties:
    • grid-column: Defines which columns an item should span. Can be specified with start and end line numbers (e.g., grid-column: 1 / 3;).
    • grid-row: Defines which rows an item should span. Can also be specified with start and end line numbers (e.g., grid-row: 2 / 4;).
    • grid-area: A shorthand property that combines grid-row and grid-column definitions.

Example


        .grid-container {
            display: grid;
            grid-template-columns: repeat(3, 1fr); /* Three equal columns */
            grid-template-rows: auto;
            gap: 10px; /* Space between grid items */
        }

        .grid-item {
            grid-column: span 2; /* Span two columns */
        }


Typography in CSS

Typography refers to the style, arrangement, and appearance of text. Good typography enhances readability and user experience on websites.

Key Concepts

  • Font Family: Defines the typeface used for text.
    • Example: font-family: 'Arial', sans-serif;
  • Font Size: Sets the size of the text. Can be specified in various units (px, em, rem, %, etc.).
    • Example: font-size: 16px; /* Fixed size
    • Example: font-size: 1.5em; /* Relative size
  • Font Weight: Controls the thickness of the font.
    • Example: font-weight: bold;
    • Example: font-weight: 700; /* Equivalent to bold
  • Font Style: Defines the style of the font, such as normal, italic, or oblique.
    • Example: font-style: italic;
  • Font Variant: Controls the usage of small-caps in text.
    • Example: font-variant: small-caps;
  • Line Height: Sets the height of lines of text.
    • Example: line-height: 1.5; /* 1.5 times the font size
  • Letter Spacing: Adjusts the space between characters in text.
    • Example: letter-spacing: 0.05em; /* Increases space between letters
  • Word Spacing: Adjusts the space between words in text.
    • Example: word-spacing: 0.1em; /* Increases space between words

Text Styling in CSS

Text styling involves modifying the appearance of text to enhance its visual impact and readability.

Text Properties

  • Text Color: Sets the color of the text.
    • Example: color: #333; /* Dark gray text
  • Text Alignment: Defines the alignment of text within an element (left, right, center, justify).
    • Example: text-align: center; /* Centers the text
  • Text Decoration: Adds decorations to text, such as underline, overline, or line-through.
    • Example: text-decoration: underline; /* Underlines the text
  • Text Transform: Controls the capitalization of text (uppercase, lowercase, capitalize).
    • Example: text-transform: uppercase; /* Converts text to uppercase
  • Text Shadow: Applies a shadow effect to the text.
    • Example: text-shadow: 2px 2px 5px rgba(0, 0, 0, 0.3); /* Adds shadow to text
  • White Space: Controls how white space inside an element is handled.
    • Example: white-space: nowrap; /* Prevents text from wrapping
  • Overflow: Specifies how to handle overflowed text.
    • Example: overflow: hidden; /* Hides text that overflows
  • Direction: Sets the text direction, useful for languages that read right to left.
    • Example: direction: rtl; /* Sets text direction to right-to-left


Colors in CSS

Colors can be specified in various ways in CSS, enabling flexibility in design.

Color Formats

  • Named Colors: CSS supports a set of predefined color names (e.g., red, blue, green).
  • Hexadecimal Colors: Colors can be represented using a hex code, starting with a # followed by six or three hexadecimal digits.
    • Example: #ff0000 (red), #f00 (shorthand for red).
  • RGB Colors: Colors can be defined using the RGB (Red, Green, Blue) model with values ranging from 0 to 255.
    • Example: color: rgb(255, 0, 0); /* Red
  • RGBA Colors: Similar to RGB, but includes an alpha value for opacity (0 to 1).
    • Example: color: rgba(255, 0, 0, 0.5); /* Semi-transparent red
  • HSL Colors: Colors can also be defined using the HSL (Hue, Saturation, Lightness) model.
    • Example: color: hsl(0, 100%, 50%); /* Red
  • HSLA Colors: Similar to HSL, but includes an alpha value for opacity.
    • Example: color: hsla(0, 100%, 50%, 0.5); /* Semi-transparent red

Backgrounds in CSS

Backgrounds can be applied to any element, allowing for enhanced design and visual interest.

Background Properties

  • background-color: Sets the background color of an element.
    • Example: background-color: #f0f0f0; /* Light gray background
  • background-image: Applies an image as the background of an element.
    • Example: background-image: url('image.jpg');
  • background-repeat: Controls the repetition of the background image. Values include:
    • repeat (default)
    • no-repeat
    • repeat-x
    • repeat-y
  • background-position: Sets the starting position of a background image.
    • Example: background-position: center; /* Center the image
  • background-size: Defines the size of the background image.
    • Example: background-size: cover; /* Cover the entire element
  • background-attachment: Determines how the background image behaves when scrolling.
    • Values: scroll, fixed, local
  • Shorthand Property: All background properties can be set using the shorthand background property.
    • Example: background: #fff url('image.jpg') no-repeat center / cover;

Gradients in CSS

Gradients allow for smooth transitions between colors and can be used as background images.

Types of Gradients

  • Linear Gradient: A gradient that transitions along a straight line.
    • Syntax: background: linear-gradient(direction, color-stop1, color-stop2, ...);
    • Example: background: linear-gradient(to right, red, yellow); /* Horizontal gradient
  • Radial Gradient: A gradient that radiates from a central point.
    • Syntax: background: radial-gradient(shape size at position, color-stop1, color-stop2, ...);
    • Example: background: radial-gradient(circle, red, yellow); /* Circular gradient
  • Conic Gradient: A gradient that rotates around a center point.
    • Syntax: background: conic-gradient(color-stop1, color-stop2, ...);
    • Example: background: conic-gradient(red, yellow, green); /* Conic gradient

Gradient Color Stops

Color Stops: Define the colors used in the gradient and their positions (optional). You can specify color stop positions using percentages or lengths.

Example: background: linear-gradient(to right, red 0%, yellow 50%, green 100%);



Responsive Design

Responsive Design is an approach to web development that ensures web pages look good and function well on various devices, including desktops, tablets, and smartphones. The primary goal is to provide an optimal user experience regardless of the device being used.

Key Principles of Responsive Design

  • Fluid Grids: Use relative units (percentages, ems, rems) instead of fixed units (pixels) for layout elements.
    • This approach ensures that elements scale proportionally based on the size of the viewport.
  • Flexible Images: Images should be responsive and scale according to the size of their containing element.
    • Use CSS to set images to max-width: 100%; to prevent them from exceeding their container’s width.
    • Example: img { max-width: 100%; height: auto; }
  • Media Queries: A key technique in responsive design that allows you to apply different styles based on the characteristics of the device.

Media Queries

Media Queries are a CSS feature that allows you to apply styles based on specific conditions, such as the width or height of the viewport. They are essential for implementing responsive design.

Syntax of Media Queries

The basic syntax of a media query is as follows:

@media (condition) { /* CSS rules here */ }

Common Conditions

  • Width: Target specific device widths.
    • Example: @media (max-width: 600px) { /* Styles for devices with a width of 600px or less */ }
  • Height: Target specific device heights.
    • Example: @media (min-height: 800px) { /* Styles for devices with a height of 800px or more */ }
  • Orientation: Apply styles based on the orientation of the device (portrait or landscape).
    • Example: @media (orientation: landscape) { /* Styles for landscape orientation */ }
  • Resolution: Target high-resolution screens.
    • Example: @media (min-resolution: 2dppx) { /* Styles for high-resolution devices */ }
  • Combining Conditions: Combine multiple conditions using and, or, or not.
    • Example: @media (max-width: 600px) and (orientation: portrait) { /* Styles for portrait orientation on small screens */ }

Best Practices for Responsive Design and Media Queries

  • Mobile-First Approach: Start by designing for the smallest screen sizes and enhance for larger screens using media queries.
  • Breakpoints: Choose breakpoints based on content needs rather than specific devices. Common breakpoints include:
    • 320px (mobile)
    • 768px (tablet)
    • 1024px (desktop)
  • Test on Multiple Devices: Regularly test your designs on various devices and screen sizes.
  • Use Relative Units: Use relative units for fonts, margins, and paddings to maintain proportionality across devices.
  • Avoid Fixed Widths: Refrain from using fixed widths for containers; use percentages or flexible units.
  • Viewport Meta Tag: Include the viewport meta tag in your HTML to control layout on mobile browsers:
    • Example: <meta name="viewport" content="width=device-width, initial-scale=1.0">


Transitions

CSS Transitions allow you to change property values smoothly over a specified duration.

Key Properties of Transitions

  • transition-property: CSS property to be transitioned.
  • transition-duration: Duration of the transition.
  • transition-timing-function: Speed curve of the transition.
  • transition-delay: Delay before the transition starts.

Example of CSS Transitions

.box {
background-color: blue;
width: 100px;
height: 100px;
transition: background-color 0.5s ease;
}
.box:hover {
background-color: red;
transform: scale(1.5);
}

Animations

CSS Animations provide more control and flexibility than transitions.

Key Properties of Animations

  • animation-name: Name of the animation.
  • animation-duration: Length of the animation cycle.
  • animation-timing-function: Animation progression.
  • animation-delay: Delay before the animation starts.
  • animation-iteration-count: Number of times to repeat the animation.

Example of CSS Animations

@keyframes slide {
0% {
transform: translateX(0);
opacity: 1;
}
100% {
transform: translateX(100px);
opacity: 0.5;
}
}
.animated-box {
width: 100px;
height: 100px;
background-color: green;
animation: slide 2s ease-in-out infinite;
}

Transformations

CSS Transformations manipulate elements by changing their size, position, and shape.

Common Transformation Functions

  • translate(x, y): Moves the element.
  • scale(x, y): Resizes the element.
  • rotate(deg): Rotates the element.

Example of CSS Transformations

.transform-box {
width: 100px;
height: 100px;
background-color: orange;
transform: rotate(45deg) scale(1.5);
}

Combining Techniques

Transitions, animations, and transformations can be combined for engaging user experiences.

Example of Combining Techniques

.box {
width: 100px;
height: 100px;
background-color: purple;
transition: transform 0.5s ease;
}
.box:hover {
transform: rotate(90deg) scale(1.2);
}

Best Practices

  • Use Sparingly: Avoid overwhelming users.
  • Performance: Use hardware-accelerated properties.
  • Accessibility: Consider users with motion sensitivity.


CSS Variables (Custom Properties)

What are CSS Variables?

CSS Variables, also known as custom properties, are entities defined by CSS authors that contain specific values to be reused throughout a document.

Syntax

CSS variables are defined using the -- prefix and can be declared within a selector.

:root {
--main-color: #3498db;
--font-size: 16px;
}

Usage

To use a CSS variable, use the var() function.

body {
background-color: var(--main-color);
font-size: var(--font-size);
}

Benefits

  • Reusability: Define once and reuse throughout the CSS.
  • Maintainability: Change a variable value in one place to update multiple elements.
  • Dynamic Changes: Variables can be changed with JavaScript, allowing for dynamic styling.

Scope

CSS variables have a scope similar to CSS selectors. Variables defined in a parent element can be accessed by child elements.

.container {
--padding: 10px;
}
.child {
padding: var(--padding);
}

Units in CSS

Types of Units

  • Absolute Units: Fixed sizes that do not change with screen size.
    Examples: px (pixels), cm (centimeters), in (inches), pt (points), pc (picas).
  • Relative Units: Sizes that are relative to other elements or the viewport.
    Examples:
    • em: Relative to the font size of the element.
    • rem: Relative to the font size of the root element.
    • %: Percentage of the parent element's size.
    • vw: Viewport width (1% of the viewport width).
    • vh: Viewport height (1% of the viewport height).
    • vmin and vmax: Relative to the smaller or larger dimension of the viewport.

Examples

Using pixels:

div {
width: 300px;
height: 200px;
}

Using percentages:

.container {
width: 80%;
}

Using rem:

h1 {
font-size: 2rem; /* 2 times the root font size */
}

Choosing Units

Absolute units are useful for fixed dimensions, while relative units are preferred for responsive design. Use relative units for font sizes, margins, and padding to enhance accessibility and responsiveness.
